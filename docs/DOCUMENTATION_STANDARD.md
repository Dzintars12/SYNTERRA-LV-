# ◉ SYNTERRA Dokumentācijas Standarts

## Mērķis

Šis dokuments definē vienotu SYNTERRA dokumentācijas standartu.

Mērķis:
- uzturēt profesionālu struktūru,
- nodrošināt LV un EN sinhronizāciju,
- saglabāt vienotu civilizācijas identitāti,
- un izvairīties no dokumentācijas haosa.

---

# ◉ Pamatprincipi

## 1. Vienota Struktūra

LV un EN dokumentācijai:
- jābūt identiskai struktūrai,
- identiskai sadaļu secībai,
- un identiskai arhitektūras loģikai.

Atšķiras tikai valoda.

---

## 2. Valodu Tīrība

### LV repozitorijs

Visam saturam jābūt latviešu valodā.

Izņēmumi:
- tehniskie nosaukumi,
- sistēmu identifikatori,
- vai starptautiski termini.

---

### EN repozitorijs

Visam saturam jābūt angļu valodā.

EN dokumentācijai jāskan dabiski starptautiskai auditorijai.

---

## 3. README Standarts

README nav pilns whitepaper.

README ir:
- ieejas portāls,
- navigācijas centrs,
- un civilizācijas atmosfēras sākumpunkts.

README jābūt:
- vizuālam,
- emocionālam,
- skaidram,
- un viegli uztveramam.

---

## 4. Progressive Information Depth

Cilvēks sākumā redz:
- virsrakstus,
- galvenos virzienus,
- un īsus paskaidrojumus.

Dziļāka informācija atveras vēlāk.

---

## 5. Three Click Navigation Principle

Lietotājam sev vajadzīgais virziens jāatrod maksimums trīs klikšķos.

---

## 6. Zero Block Principle

ZERO BLOCK vienmēr paliek atvērts jaunām idejām.

Tas nekad netiek aizņemts.

---

## 7. Fraktālā Attīstība

Katrs:
- Block,
- projekts,
- vai kopiena

var radīt nākamos attīstības virzienus.

---

## 8. Civilization Atmosphere

SYNTERRA dokumentācijai:
- jābūt mierīgai,
- dziļai,
- strukturētai,
- un dzīvai.

Mērķis nav radīt informācijas troksni.

Mērķis ir radīt dzīvas civilizācijas sajūtu.
